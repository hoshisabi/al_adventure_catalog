# CLAUDE.md — Adventure Catalog Project Guide

## Project Overview

This is a **sortable/filterable web catalog of Adventurers League (AL) adventures**, hosted at https://hoshisabi.com/al_adventure_catalog/. The site is static HTML/JS; Python scripts under `maintaindb/` scrape and process DM's Guild product data into JSON files consumed by the frontend.

**TASKS.md is the active task list and communication file** — use it to track progress, note blockers, and coordinate across sessions.

---

## CRITICAL: Windows PowerShell Environment

This project runs on **Windows with PowerShell**. The system prompt says "use Unix shell syntax" but the project itself requires PowerShell for most commands. Use PowerShell equivalents, not Unix commands:

| Unix | PowerShell |
|------|------------|
| `grep` | `Select-String -Pattern "..." file` |
| `cat` | `Get-Content file` |
| `head -N` | `Select-Object -First N` |
| `find` | `Get-ChildItem` |

- Always run Python scripts from **project root** with `uv run python -m maintaindb.<script>`
- Never `cd` into `maintaindb/` to run scripts — relative imports break
- For `jq` with complex filters, write to a temp `.bat` file to avoid quoting issues
- Use `Test-Path` to check if files exist before operations

```
├── TASKS.md                  ← roadmap and task tracking

├── CODE_RECOGNITION.md       ← documents how adventure codes are parsed
├── AI_ASSISTANT_GUIDELINES.md ← Windows/PowerShell command reference
└── TESTING.md                ← test setup instructions
```

## Running Scripts

All from **project root**:

```powershell
uv run python -m maintaindb.aggregator          # rebuild catalog.json
uv run python -m maintaindb.process_downloads   # parse HTML in dmsguildinfo/ → _dc/
uv run python -m maintaindb.process_rss --url <url>  # fetch RSS → _dc/
uv run python -m maintaindb.stats               # print catalog statistics
uv run python -m maintaindb.generate_fixup_html # generate fixup.html for missing data
uv run pytest                                   # run tests
```

Convenience scripts:
- `lp.bat` — serve site locally
- `lp.bat rebuild` — regenerate catalog.json then serve

## Data Workflow

1. **Ingest**: RSS parser (`process_rss.py`) or HTML bookmarklet + `process_downloads.py`
2. **Review**: Check `_dc/*.json`; use `generate_fixup_html.py` to find missing fields
3. **Aggregate**: `aggregator.py` → `assets/data/catalog.json`
4. **Test**: `uv run pytest`

RSS entries are tagged `needs_review: true` because the feed lacks authors, hours, tier, APL, level_range.

## catalog.json Format (minified, key-abbreviated)

See `CATALOG_FORMAT.md` for full reference. Short version:

| Key | Field | Notes |
|-----|-------|-------|
| `i` | product_id | String |
| `n` | title | |
| `c` | code | e.g. `DDAL09-01` |
| `a` | authors | Array |
| `s` | season | |
| `h` | hours | String, e.g. `"2-4"` |
| `t` | tier | Integer 1–4 |
| `d` | date_created | YYYYMMDD |
| `p` | campaigns | Bitmask (1=FR, 2=Eberron, 4=Ravenloft, 8=Dragonlance) |
| `f` | flags | Bitmask (1=community, 2=dungeoncraft, 4=salvage) |

## Critical Warnings

- **Never delete files in `maintaindb/_dc/`** without a backup — some contain manually entered data that cannot be regenerated.
- **Never commit DM's Guild HTML** to the public repo — it belongs in the private `private-fixtures/` submodule.
- `adventure_extractors.py` and `adventure_normalizers.py` are **unused refactoring drafts** — don't wire them in without checking `REFACTORING_TASKS.md`.

## Architecture Notes

- `adventure.py` (`DungeonCraft` class) is the **active** parsing library.
- Component/bundle files: products with multiple adventures use `{product_id}-{N}.json` naming (e.g. `545950-01.json`). See `BUNDLES_AND_COMPONENTS.md`.
- URL derivation: `https://www.dmsguild.com/product/{base_id}/?affiliate_id=171040` (strip trailing `-N` suffix from product_id).
- Adventure code parsing lives in `adventure_utils.py` (`get_adventure_code_and_campaigns`). See `CODE_RECOGNITION.md`.
